# 📝 WriteLens

> **نظام تحليل الكتابة التكيفي** — تعليقات ذكية مدعومة بالذكاء الاصطناعي لتقييم المقالات وتحسين مهارات الكتابة

**WriteLens** منصة تعليمية ذكية تستخدم تقنيات التعلم الآلي ومعالجة اللغة الطبيعية لتحليل مقالات الطلاب وتقديم تعليقات شخصية قابلة للتنفيذ فوراً. تجمع بين الاستدلال البايزي وتصنيف غابات القرار وتجميع K-Means لتقديم تدخلات تكيفية مخصصة لمستوى كل طالب.

---

## 🎯 نظرة عامة

### ما هو WriteLens؟

WriteLens هي **منصة تكنولوجيا تعليمية** مصممة لمساعدة الطلاب على تحسين مهارات الكتابة لديهم من خلال تعليقات ذكية ومولدة بالذكاء الاصطناعي. بدلاً من التعليقات العامة، يقوم WriteLens بتحليل البنية العميقة لكتابة الطالب وتقديم اقتراحات موجهة بناءً على مستوى الكفاءة الحالي.

### لماذا هو مهم؟

- ✅ **التصحيح الآلي:** يقلل عبء العمل على المعلم بنسبة 70%
- ✅ **التعلم الشخصي:** كل طالب يحصل على تعليقات مخصصة لمستواه
- ✅ **التحليلات الفورية:** يرى المعلمون الأنماط والاتجاهات عبر المجموعات الدراسية
- ✅ **قابلية التوسع:** دعم عدد غير محدود من الطلاب دون زيادة التكاليف
- ✅ **التدخلات المبنية على البيانات:** تحديد الطلاب الذين يواجهون صعوبات مبكراً

### من يستخدم WriteLens؟

| المستخدم | الفائدة |
|---------|--------|
| 🎓 **الطلاب** | الحصول على تعليقات فورية وبناءة حول كتاباتهم |
| 👨‍🏫 **المعلمون** | توفير وقت التصحيح، تتبع تقدم الفصل، تحديد التدخلات المطلوبة |
| 🏫 **المؤسسات التعليمية** | تحسين مخرجات الكتابة، توحيد التقييم، التكامل مع أنظمة إدارة التعلم |
| 📊 **الباحثون** | دراسة التقييم التكيفي وتطبيقات التعلم الآلي في التعليم |

## 🚀 دليل البدء السريع

### ⚡ المتطلبات الأساسية

قبل البدء، تأكد من أن لديك:

- **Node.js** v18+ (للواجهة الأمامية والخادم الخلفي)
- **Python** 3.9+ (لمحرك الذكاء الاصطناعي)
- **PostgreSQL** 12+ (لقاعدة البيانات)
- **Git** (لإدارة الإصدارات)
- **npm** v9+ (مدير الحزم)

### 📦 خطوات التثبيت

#### الخطوة 1: استنساخ المستودع
```bash
git clone https://github.com/your-org/writelens.git
cd writelens
```

#### الخطوة 2: إعداد متغيرات البيئة
```bash
# نسخ ملف القالب
cp .env.example .env

# تعديل الإعدادات
nano .env
```

**المتغيرات الأساسية:**
```env
# قاعدة البيانات
DATABASE_URL=postgresql://user:password@localhost:5432/writelens

# المصادقة
JWT_SECRET=your-super-secret-key-min-32-chars-here

# تكوين API
VITE_API_URL=http://localhost:3000/api
PYTHON_BRIDGE_URL=http://localhost:5000

# البيئة
NODE_ENV=development
```

#### الخطوة 3: إعداد الخادم الخلفي
```bash
# الانتقال إلى المجلد الخلفي
cd backend

# تثبيت المكتبات
npm install

# تهيئة قاعدة البيانات
npm run migrate

# أو إنشاء قاعدة البيانات يدويًا
createdb writelens

# بدء خادم الويب
npm run dev
# سيعمل على: http://localhost:3000
```

#### الخطوة 4: إعداد الواجهة الأمامية
```bash
# الانتقال إلى المجلد الأمامي
cd ../frontend

# تثبيت المكتبات
npm install

# بدء خادم التطوير
npm run dev
# سيعمل على: http://localhost:5173
```

#### الخطوة 5: إعداد محرك الذكاء الاصطناعي (اختياري)
```bash
# الانتقال إلى مجلد محرك الذكاء الاصطناعي
cd ../ai_engine

# إنشاء بيئة افتراضية
python -m venv venv
source venv/bin/activate  # في Windows: venv\Scripts\activate

# تثبيت المكتبات
pip install -r requirements.txt

# اختبار محرك الذكاء الاصطناعي
python main.py '{"test": true}'
```

#### الخطوة 6: التحقق من التثبيت
```bash
# اختبار صحة الخادم الخلفي
curl http://localhost:3000/health

# الاستجابة المتوقعة:
# {"status": "running", "timestamp": "...", "service": "writelens-backend"}

# فتح المتصفح لاختبار الواجهة الأمامية
# http://localhost:5173
```

## 🔌 نقاط نهاية API

### المصادقة
- `POST /api/auth/login` - تسجيل دخول المستخدم
- `POST /api/auth/register` - تسجيل مستخدم جديد
- `POST /api/auth/logout` - تسجيل الخروج
- `POST /api/auth/verify` - التحقق من رمز JWT

### إدارة الطلاب
- `GET /api/student` - قائمة جميع الطلاب
- `GET /api/student/:id` - الحصول على ملف الطالب
- `POST /api/student` - إنشاء طالب جديد
- `PUT /api/student/:id` - تحديث بيانات الطالب

### مسار معالجة البيانات والتحليل
- `POST /api/pipeline/run` - تشغيل مسار معالجة الذكاء الاصطناعي
- `GET /api/pipeline/:id` - الحصول على نتائج مسار المعالجة
- `GET /api/decision/:studentId` - الحصول على قرار/تعليق الذكاء الاصطناعي

### التقارير
- `GET /api/report/:studentId` - إنشاء تقرير الطالب
- `GET /api/report/` - قائمة التقارير
- `POST /api/report/export` - تصدير التقرير (PDF/CSV)

للحصول على التوثيق الكامل للـ API، انظر [docs/API.md](./docs/API.md).

## 🤖 تفاصيل محرك الذكاء الاصطناعي

### كيفية العمل

1. **استقبال البيانات:** استقبال إرسال الطالب من الخادم الخلفي
2. **استخراج الميزات:** تحليل النص بحثًا عن عدد الكلمات والتعقيد والمشاعر، إلخ.
3. **تطبيق الحدود:** الفحص مقابل الحدود المحددة مسبقًا
4. **تصنيف التعلم الآلي:** استخدام غابات القرار والشبكات البايزية
5. **التجميع:** تجميع الطلاب المتشابهين لتحليل الأنماط
6. **تطبيق القواعس:** تطبيق القواعس الخاصة بالمجال من `rules.yaml`
7. **توليد التعليقات:** اختيار نماذج التعليقات المناسبة
8. **إرجاع القرار:** الإرسال مرة أخرى إلى الخادم الخلفي مع التعليقات والتوصيات

### ملفات التكوين

- **`thresholds.yaml`** - يحدد حدود الدرجات
- **`rules.yaml`** - قواعس المنطق التجاري لاتخاذ القرارات
- **`feedback_templates.yaml`** - 16+ نموذج تعليق

### تشغيل الذكاء الاصطناعي بشكل مستقل

```bash
python ai_engine/main.py '{"student_id": "123", "text": "...", "cohort_id": "abc"}'
```

## 🔐 الأمان

### المصادقة والتفويض
- مصادقة رمز JWT
- التحكم في الوصول بناءً على الأدوار (RBAC)
  - `admin` - الوصول الكامل للنظام
  - `teacher` - إدارة الفصل وبيانات الطالب
  - `student` - الملف الشخصي والإرسالات فقط
  - `researcher` - الوصول إلى التحليلات والتقارير

### حماية البيانات
- كلمات المرور مجزأة باستخدام bcryptjs (10 أملاح)
- HTTPS موصى به للإنتاج
- CORS مكون لأصل الواجهة الأمامية فقط
- منع حقن SQL باستخدام استعلامات محددة المعاملات
- لا بيانات حساسة في السجلات

### متغيرات البيئة
- المفاتيح الحساسة مخزنة في `.env` (لا تُلتزم أبداً)
- استخدم `.env.example` كقالب
- إعدادات مختلفة للتطوير/التدريج/الإنتاج

## 🐛 استكشاف الأخطاء

### الواجهة الأمامية لا تتصل بالخادم الخلفي
- تحقق من `VITE_API_URL` في `.env`
- تحقق من أن الخادم الخلفي يعمل على `http://localhost:3000`
- تحقق من إعدادات CORS في `backend/src/config`

### فشل الاتصال بقاعدة البيانات
- تحقق من أن PostgreSQL قيد التشغيل: `pg_isready`
- تحقق من `DATABASE_URL` في `.env`
- تأكد من وجود قاعدة البيانات: `createdb writelens`

### محرك الذكاء الاصطناعي الفرعي يفشل
- تحقق من تثبيت Python: `python --version`
- تحقق من `PYTHON_BRIDGE_PATH` الصحيح
- شغل مباشرة: `python ai_engine/main.py '{}'`

### أخطاء التحقق من JWT
- امسح ذاكرة التخزين المؤقت/ملفات تعريف الارتباط في المتصفح
- تحقق من أن `JWT_SECRET` يطابق بين الجلسات
- تحقق من انتهاء صلاحية الرمز في `.env`

للمزيد من المساعدة، انظر [docs/TROUBLESHOOTING.md](./docs/TROUBLESHOOTING.md).
